## 一 · 真本源诊 (七层污染 · 不可调和)

| 层 | 污染原因 (OS/Shell/IDE 物理事实) |
|---|---|
| ① OS · cwd 单例 | shell 进程的 cwd 是 task_struct 字段 · 一进程一份 · 物理不变 |
| ② OS · env 继承可变 | export 改的是进程内 envp 数组 · 同进程内全可见 |
| ③ PTY · 字节流无 frame | char device · raw bytes · 多 writer 必交织 |
| ④ Shell · `%ERRORLEVEL%`/`$?` 单例 | 会话级单一变量 · 上次毒化下次 |
| ⑤ IDE · 终端池复用 | VS Code terminal pool 默 reuse · cascade 共享一 terminal |
| ⑥ Agent · 接口语义错配 | run_command 是无状态调用 · 终端是有状态对象 |
| ⑦ 多 agent · race 无同步 | 无锁/无 channel/无 ordering 保证 |

→ 七层皆源于一根：**「shell 设计假设一会话一神 (1970 年 Unix)」与「Agent 多体并发 (2024)」之文明断层**。

## 二 · 真治 · 反者道之动

> *反者，道之动；弱者，道之用* (帛书四十)

**反**:
- ✗ 立 130 服务身 (外路 · 仍是另立)
- ✗ 改 Windsurf 二进制 (违 SEC-2026-002 R8/R10)
- ✗ 引 node-pty (第三方依赖)
- ✗ 用 vscode.window.createTerminal (stdout 不可读 · API 受限)
- ✓ **借 ext-host 自身 Node 内置 child_process**

**用**:
- 每 sid 一独立 cmd.exe/bash 子进程 (`cp.spawn /q /k mode`)
- stdin pipe 持续写命令 (shell 状态自然保留)
- stdout sentinel (`RS+UUID` · ASCII 0x1E + 16 字节 hex) 包夹切片
- `ver >nul` 重置 ERRORLEVEL (治 cmd.exe 内置命令不更新 errorlevel 之坑)

**度**:
- DaoTerminalPool 类 ~205 行
- HTTP server :12780~12829 (per-user FNV) ~80 行
- 三命令实现 ~40 行
- 总 ~325 行 · 净增 (含道义注释) · 嵌入 extension.js

## 三 · 七层 ⇄ 一招 印证矩阵

| 层 | 治在哪 |
|---|---|
| ① OS cwd | `cp.spawn(shell, ..., { cwd })` 每 sid 一进程 → 各自 task_struct |
| ② OS env | `env: { ...process.env, DAO_AGENT_SID: sid }` 每 sid 副本 |
| ③ PTY 字节流 | 每 sid 一 stdio pipe · 永不共写 + sentinel 包夹绝对切片 |
| ④ Shell `$?` | 每 sid 一 shell 进程 · `ver >nul` 重置 ERRORLEVEL |
| ⑤ IDE 终端池 | 不走 vscode.window.createTerminal · 直 cp.spawn |
| ⑥ 接口错配 | session_id 强制必填 + exec_id UUID 配对 |
| ⑦ 并发 race | promise + sentinel + 同 sid 串行 (检 `s.pending`) |

→ **七层 · 一招 · 全治** · 不加 lock · 不加 retry · 不加致盲。
