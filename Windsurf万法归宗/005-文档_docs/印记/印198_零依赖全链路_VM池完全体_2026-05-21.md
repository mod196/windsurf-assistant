## 核心突破

### 零GitHub依赖部署（spawner v1.9.0）

**之前（v1.8.0）**：spawner在Phase2通过curl从GitHub下载server.js
- 依赖GitHub网络可达
- 依赖repo公开或PAT
- VM网络不通时失败

**现在（v1.9.0）**：启动时将server.js压缩内嵌到spawner内存
```
[spawner] engine embedded: 81137B raw → 22841B gz → 30456C b64
```
Phase2 bash直接解码内嵌引擎，无需任何网络：
```bash
cat << 'ENGINEB64EOF' | base64 -d | gunzip > ~/app/engine.js
H4sIAAAA...（22KB gzip+base64）
ENGINEB64EOF
# 如果内嵌失败，自动降级到URL下载
```

## 新增功能

### server.js v2.3.0

#### devinChat — Devin Cloud模型直接调用
```
model: "devin"                    → Devin自选模型执行任务
model: "devin/claude-opus-4-5"   → 指定模型执行
model: "devin/gpt-4o"            → GPT-4o通过Devin
```
工作流：
1. 消息 → 构建任务提示词
2. POST /v1/sessions（Devin Cloud）
3. 轮询状态（5s间隔，最多120s）
4. 提取assistant消息返回

#### 全模型列表（/v1/models）
- Cascade: 30+模型（Claude/GPT/Gemini/SWE/DeepSeek/Grok...）
- Devin Cloud: 11个devin/model-name变体
- External(BYOK): 动态注册
- 返回pools状态信息（cascade/devin/ws/external）

#### watchdog.sh 自持久化
引擎启动时自动写入~/app/watchdog.sh并后台执行：
- 每15秒检查health
- 进程崩溃自动重启
- 使用lock文件防止重复

#### PM2持久化（spawner v1.9.0）
Phase2自动安装pm2并注册开机自启：
```bash
pm2 start ~/app/engine.js --name dao-vm
pm2 save --force
pm2 startup | grep sudo | bash
```

### dao_devin_proxy.js v1.0.0（新文件）

本地HTTP代理，拦截Devin API调用路由到VM池：

```
浏览器HTTP代理: http://127.0.0.1:3457
    ↓
dao_devin_proxy (:3457)
    ├── api.cognition.ai → localhost:8888 (VM池)
    ├── app.devin.ai/api/* → localhost:8888 (VM池)  
    └── 其他 → 直连原始服务器
```

使用方式：
1. 设置浏览器HTTP代理为 `http://127.0.0.1:3457`
2. 打开 https://app.devin.ai — AI调用自动经过VM池
3. 或设置环境变量：`HTTP_PROXY=http://127.0.0.1:3457`
