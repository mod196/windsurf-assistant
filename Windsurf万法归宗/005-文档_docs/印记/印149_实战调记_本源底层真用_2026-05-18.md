## 二 · 问题谱 6 件 · 真本源破 · 解之径

### § 2.1 · 问题 1 · sonnet4.6t 长输出失败 (完 tokens=0 · 仅返 10 字)

**症**: `sonnet4.6t` chat 写 TODO · 55s 返「道·待办，已成。」10 字 · `completion_tokens=0`

**探**: full response 之 `x_dao.engine="devin-cloud"` · sessionId=`devin-07c228...`

**真本源破**: sonnet4.6t 走 **Devin ACP agentic 路径** · 非普通 chat completion。Devin 在云端 VM 真做事 (创文件/跑命令/git)，`.content` 仅返「meta 对话回复」(如「已创建完成」)。

**解之径** ✅ (印 149 § 4.3 实证):
- system prompt 加: `直接给完整代码 · 不解释 · 不调工具 · 不创文件 · 仅返代码字符串包于 ```html``` 内`
- user prompt 末加: `仅返代码 · 不调任何工具 · 不操作文件 · 直接输出字符串`
- 二条 prompt 引导 Devin 不进入 agentic flow · 退回 chat 模式 · **14.3s 返 39318 字 + 91 行真 HTML**

### § 2.2 · 问题 2 · windsurf-* cascade 502/503

**症**: 本机 :7781 之 `windsurf-sonnet/opus/gpt-5` chat → 502 Bad Gateway (1.6s 立刻返)

**探**: `/health` 之 windsurf 节: `cooldown_keys=3 · ok_total=0 · err_total=3 · chat_status="Free tier 当下 quota 全耗尽 · 待 reset"`

**真本源破**: Cascade 之 ConnectRPC ws 是 **server.codeium.com 之 Free tier IP-based quota**·主公本机一 IP 之 quota 已耗尽 (5 件 key 全 cooldown)。

**解之径** ⏳:
1. 等 Free tier reset (~1h or daily 周期)
2. 起 **更多 N VM** 让每件 VM 之独立 IP 各拿独立 quota (但 VM 是空 Devin · 无 windsurf 装态 · 见问题 6)
3. 主公本机重启 Windsurf · 触 cascade 重连
4. 换 IP (cf tunnel / 代理) 暂避 quota
