## 修复清单

### B-new · pickRoute 直连模式路由选择

**文件**: `04-Hub/dao_hub_orchestrator.js` line 932

**问题**: 原代码 `gatewayRoutes.filter(r => r.healthy && r.vmUrl && ...)` 要求 `r.vmUrl` 存在才进入主路由池。但直连模式下所有路由 `vmUrl = null`，始终走 fallback → 只取第一个账号 → "Tried 1 account"。

**修复**:
```js
// 修复前 (Bug):
const healthy = gatewayRoutes.filter(r => r.healthy && r.vmUrl && !excludeEmails.has(r.email));

// 修复后 (B-new):
const healthyVM = gatewayRoutes.filter(r => r.healthy && r.vmUrl && !excludeEmails.has(r.email));
if (healthyVM.length > 0) { /* VM模式 round-robin */ }
// 直连模式 fallback:
const healthyDirect = gatewayRoutes.filter(r => r.healthy && !r.vmUrl && !excludeEmails.has(r.email));
if (healthyDirect.length > 0) { /* 直连 round-robin */ }
```

**效果**: Chat 从 "Tried 1 account" → "Tried 20 accounts" (MAX_RETRIES=20)，正确轮转所有健康路由。
