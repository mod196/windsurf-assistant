## 一、致命真相 (TL;DR)

**Windsurf 1.110.x 的 bundled summarizer agent 启动机制存在进程泄漏 bug**：

每次 Cascade 调用 summarizer，就启动一对：
```

```

**调用结束后两个进程都不回收**，累积成僵尸。

## 四、根因解构（道法自然）

### 🔥 P0 · ACP summarizer 进程泄漏（今日新生）

**调用链证据**：
```
bash.exe path: C:\Program Files\Git\bin\..\usr\bin\bash.exe
bash cmd: bash --login -c "'/e/Windsurf/.../devin.exe' 'acp' '--agent-type' 'summarizer'"
devin.exe path: E:\Windsurf\resources\app\extensions\windsurf\devin\bin\devin.exe
devin cmd: devin.exe acp
devin parent: PID=62404  (Windsurf renderer)
```

**机理**：
1. Cascade 主对话中需要"总结对话历史" / 切上下文窗口时，按 ACP 协议召唤 summarizer 子代理
2. Windsurf renderer (PID=62404) → spawn git bash → bash 中再 spawn devin.exe
3. summarizer 完成总结、把结果传回 Cascade
4. **bug**：bash 进程不 exec 等 devin 退出但又不主动 wait → bash 退而不死、devin 退而不死
5. Windsurf 视之"成功"，但 Windows 仍保留 zombie 进程占有 PID/handle
6. 累积 73h 后：3722 zombies @ 141；17h 后：413 zombies @ 179

### P1 · 反代服务未启（历史遗留）
5/15 注入了 `127.0.0.1:8878`，但 010-反代_Proxy 未起服。所有 Cascade 流量被推到拒绝连接 → 退到官方 URL → 走代理 → 慢。

### P2 · 双机皆未重启 Windsurf
5/15 治理后 settings + 扩展已修，但 Windsurf 主进程**未冷启**——治理只生效在 settings 文件层，运行时仍走旧状态。

### P3 · LS 又开始膨胀
141 上 PID 233180 启动 11 分钟已吃 2.7GB RAM——同样的 04-25 之病再现。

## 五、处方（最小动作集 · 按 ROI）

### 🚀 急救一波（5 秒，不重启系统）

**A1 · 141 上批量 kill devin/bash 僵尸**（释放 ~3722 进程槽 + 数 GB 内存）

```powershell
# 通过 WinRM 远程执行
Invoke-Command -ComputerName 192.168.31.141 -Credential $cred -Authentication Negotiate -ScriptBlock {
  # 只杀 ACP summarizer 路径，避免误杀用户的 git bash
  Get-CimInstance Win32_Process -Filter "Name='devin.exe'" | Where-Object { $_.CommandLine -match 'acp' } | ForEach-Object { Stop-Process -Id $_.ProcessId -Force -EA 0 }
  Get-CimInstance Win32_Process -Filter "Name='bash.exe'"  | Where-Object { $_.CommandLine -match 'devin\.exe' } | ForEach-Object { Stop-Process -Id $_.ProcessId -Force -EA 0 }
  Write-Host "Cleanup done. Remaining: devin=$(@(Get-Process devin -EA 0).Count) bash=$(@(Get-Process bash -EA 0).Count)"
}
```
