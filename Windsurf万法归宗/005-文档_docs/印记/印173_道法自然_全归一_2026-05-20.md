## 四、根因分析 (WS Pool 全部失效)

```
根因: tokens_ws_59.txt 的 23 个 sk-ws-01-* token
      全部 resource_exhausted (quota耗尽)
      
表现: 每次推理 HTTP200 但返回空内容
      fleet 误判 ok++ fail++ (双计数)
      fallback → :7780 ACP wss → timeout (15s)
      → 整个推理链路断路
      
修复路径:
  1. node ws_bulk_auth.js --n 60  ← 正在运行中
     → 从 244 账号获取新 auth1_* tokens → tokens_ws_all.txt
  2. curl :7790/fleet/reload-ws   ← ws_bulk_auth 完成后调
     → 热重载新 tokens · 即刻可用
  3. spawn_all.js --n 3           ← 正在运行中
     → 3件新VM (devinapps.com) → VM Pool B 恢复
```
