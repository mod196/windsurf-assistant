## 六 · 真症 + 修

### 6.1 fleet_master refreshFleet 不回写 vm_pool.json

**症**: vm_omni 写入时 status=alive · 但 cf-tunnel 死后 status 字段不更新 · vm_pool.json 显 `total=11 alive=0` 假态。

**修 (印 161)**: 手工清旧 11 件 · 留 idx=0 真活。后续 fleet_master 加 timer 自动 refreshFleet (本印未做 · 留印 162)。

### 6.2 PowerShell ConvertTo-Json 单元素数组之坑

**症**: `,@($vm) | ConvertTo-Json` 把单元素 array 写成单对象 · vm_pool.json 不再是 array · fleet_master parse 失败。

**修**: 用 Node 之 `JSON.stringify(Array.isArray(j)?j:[j])` 强保 array 形。

### 6.3 SP `replace` 非法 · 应 `custom`

**症**: 我用 `strategy:replace` (习惯名) · dao_proxy 七态无此 · 返 400。

**修**: 改 `custom`。dao_proxy 七态: `bypass/override/prepend/append/dao/custom/usernote`。

### 6.4 LLM 不严格遵循 customSp

**症**: 设 `customSp="只用文言文，不超 20 字"` · LLM 返大段现代汉语。

**真因**: Devin agent 内的提示词更强 · 部分覆盖了 customSp。这是 LLM 行为细节 · 非链路问题。链路接口 100% 真通。

**改善**: 用 `dao` strategy (注 7204 字帛书+阴符经全文) 影响更深 (本印未做 · 留主公兴时验)。
