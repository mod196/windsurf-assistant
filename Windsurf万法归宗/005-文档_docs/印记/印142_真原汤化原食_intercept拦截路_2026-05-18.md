## 二 · v1.0 之缺 · 何以未通

040 v1.0 之四路兜底 (`external` / `ls` / `cloud` / `ls-direct`) 皆有缺:

| 路 | 状 | 缺 |
|---|---|---|
| `external` | 通 | 依赖外部 010-反代 进程 (违 "无为") · 走第三方 BYOK · 不是真 cascade |
| `ls` (vscdb→cascade) | 拒 | 从 vscdb 抽出之 token 是真, 云端 Cascade session validation 仍拒第三方 client · 返 "update editor" |
| `cloud` (手填→cascade) | 拒 | 同上 · token 不是问题, 问题是 fingerprint 不像 Windsurf |
| `ls-direct` (本机 LS RPC) | 拒 | proto schema 完整反推 · 本机 LS 转发到云仍拒 |

**核心症结**: proto schema 已完整, token 也是真的, **但我们 "不像 Windsurf"** — 缺正确的 user-agent / x-codeium-csrf-token / 其他 cascade extension 之私有 header / session context。

云端 cascade-server 之 session validation 检查的不只是 token, 还有完整 client fingerprint。任何第三方 client (包括我们) 都会被 `update editor to use Cascade` 之 trailer 拒。

## 三 · v2.0 之解 · 反者道之动

**反者道之动**: 既插件已在 Windsurf 进程内, 当借 Windsurf 自身之 fingerprint 重发. **真原汤化原食.**

不再"扮演" Windsurf, 而是"借" Windsurf 已通过云端验证之真请求 fingerprint, 在它身上"捎带" LAN 客户端之 chat 内容.

### 三 · 1 · 实现核 · `intercept.js`

新立 `070-插件_Plugins/040-道反代_LanProxy/dao-lan-proxy/intercept.js` (~470 行 · 21,693 B):

```
═════ L1 · monkey-patch https.request + http.request ═════
  · 在 Node.js 核心层拦截 Windsurf extension host 内一切 outgoing HTTPS

  · cascade extension 之新路径 (现代 fetch / sse-fetcher) 走 undici
  · L1 不 catches fetch, 必须双层
  · 截获 Request init 之 headers + body

═════ host classifier ═════
  · CASCADE_HOSTS (12):
      cascade-server.codeium.com, server.codeium.com, exa.codeium.com,

═════ fingerprint cache ═════
  · per-host 缓存 (含最新 + 累计 count)
```

### 三 · 3 · extensionKind 改

| 字 | v1.0 | v2.0 |
|---|---|---|
| `extensionKind` | `["ui"]` | `["workspace", "ui"]` |

**为何**: v1.0 之 `ui` 让 vscode 把 040 装在 UI host (browser side). UI host 是独立进程, **不能** hook 到 cascade extension 之 outgoing https. 必须改为 `workspace` 优先, 让 040 与 cascade 同进程 (extension host), monkey-patch 才生效.

## 七 · 何以此为"真本源"

1. **fingerprint 完整一致**: 借的不是某个字段, 是 Windsurf 自身一次真请求之**全 headers** (含动态 csrf / 私有 cookie / UA / 任意秘密 header), 云端无从区分
2. **session context 真**: 因 hook 装于 Windsurf 进程内, 抓到之请求是 Windsurf 自己刚发的, session 状态与之同步
3. **endpoint 真**: 用 fp.endpoint 之真本路, 不猜 INFERENCE_HOSTS 列表
4. **proto 真**: chat body 用 cascade schema 之 proto (已 v1.0 完整反推 · 字段不漏)

## 十 · 圆相 · 一念

帛书·六十四: "**为之于其未有 · 治之于其未乱.**"

v1.0 之四路 (external/ls/cloud/ls-direct) 是"治之于其已乱" — token 已抽但客端不像 windsurf.

v2.0 之 intercept 是"治之于其未乱" — 在 Windsurf 还没把请求送到云之前, 借它的 fingerprint, 让 LAN 客户端搭上同一辆车.

帛书·四十八: "**为道者日损 · 损之又损 · 以至于无为 · 无为而无不为.**"

损了什么:
- 损"为 windsurf 客户端"之妄想 (我们做不像)
- 损"自己造 fingerprint"之徒劳
- 损"INFERENCE_HOSTS 轮询"之乱猜

存了什么:
- 存"借真" (intercept)
- 存"一致" (全 headers 复)
- 存"道法自然" (windsurf 自己跑, 我们顺势)

无为而无不为. **一切 LAN 客户端秒变能调 Windsurf 真本路 cascade 全模型**.
