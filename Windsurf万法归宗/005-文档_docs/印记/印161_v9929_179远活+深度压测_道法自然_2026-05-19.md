## 三 · 关键诊 · 「kill ext-host vs windsurf --new-window」之微妙

### 错路一 (我先试)
```
kill ext-host 53368 → respawn 56348 → ✗ dao 未 activate · 仅 lang-server 起
```

### 因
> Windsurf ext-host crash 后 main 自动 respawn ext-host, **但 workbench 不一定立即 dispatch 全 ["*"] activation events**.
> 56348 起后只激活了「on-the-fly 触发」的扩展 (markdown / json / tsserver / eslint · 因 ext file open trigger), dao-proxy-min ["*"] 未触.

### 真路 (反者道之动)
```
windsurf.cmd --new-window
  → main 18104 收 cli IPC
  → 开新 workbench window
  → 新 window 自动 fresh-attach 新 ext-host (60268/59800)
  → workbench 真 dispatch ["*"] events
  → dao-proxy-min v9.9.29 activate
  → _startDaoTermService(ctx) 起
  → http :12817 真活
```

### 道义
> *反也者，道之动也* (帛书四十)
>
> 反 — 不必 kill main 18104 (破坏一切) · 不必 RDP 手动 reload (需 GUI) · 反借 cli `--new-window` 之轻路
>
> *弱者，道之用也* (帛书四十)
>
> 弱 — 不动现窗 · 不破现态 · 只加一新 window · ext-host 自然 fresh-activate
>
> *为之于其未有，治之于其未乱* (帛书六十三)
>
> 治 — kill + respawn 是治已乱 (慢且不全) · --new-window 是治未乱 (顺势而起)

## 四 · FNV 算法跨语言坑 (印 161 真补)

### 症
e2e 脚本 `Get-DaoTermPort` 给主公 expected_port 显示错误 (12780 实为 12794) · 不影响测试 (因 fallback 扫端口) · cosmetic 但碍真凭.

### 因
```powershell
[bigint]0xFFFFFFFF
```
PS 解析顺序:
1. `0xFFFFFFFF` 字面 → 默 **[Int32]** · 但 4294967295 超 Int32::MaxValue (2147483647)
2. PS 自动 overflow 处理 → 给 **Int32(-1)** (二补码 32 位全 1)
3. `[bigint](-1)` → BigInt(-1) → 数学无限二补码 = 全 1
4. `BigInt -band -1` = **identity (无 mask)**!

### 真凭 (Node + PS 对比)
```
Node Math.imul administrator → h_signed=-826333814 (0xCEBF258A) port=12794
PS BigInt    administrator → h_u=180792340... (huge) port=12784  ← 错
                ↓ 修 ↓
PS BigInt    administrator → h_u=3468633482  (0xCEBF258A) port=12794  ← 对
```

### 治
```powershell
# 错
$mask32 = [bigint]0xFFFFFFFF              # = -1 (Int32 overflow)
# 对
$mask32 = [bigint]4294967295              # 十进制显式
# 或
$mask32 = [bigint][uint32]0xFFFFFFFF      # uint32 cast
# 或
$mask32 = [bigint]'0xFFFFFFFF'            # 字符串解析 (BigInt 自识 hex)
```

### 道义
> *天下之难作于易，天下之大作于细* (帛书六十三)
>
> 一个十六进制字面 · 跨语言语义不同 · 真治在「与 Node Math.imul 1:1 一致」之细

## 六 · 用户名两源之微妙 (印 161 实察)

> 主公本机 USERNAME=`Administrator` · 但实活 :12809 = `zhou` FNV port

```
源 ①: PowerShell $env:USERNAME    → "Administrator"  (Windows env var)
源 ②: Node os.userInfo().username → "zhou"           (OS user 真名)

→ Get-DaoTermPort(Administrator) = 12794
→ Get-DaoTermPort(zhou)          = 12809 (实活)
→ 两者差 15 端口
```

### 因
本机 admin 用户 `zhou` (账号实名) · 但作 Administrator 组成员有 elevated 时 `$env:USERNAME` 显 `Administrator` (Windows "Run as Administrator" 行为).

`os.userInfo()` 走 SID → 用户对象 → 真 username `zhou` · 无关 elevation.

### 不变契
- ext.js 内 v9.9.29 用 `os.userInfo().username` → 永真 `zhou`/`zhouyoukang`/...
- e2e 脚本 `Get-DaoTermPort($env:USERNAME)` 显 expected → 不一定真 (在 elevated 上下文)
- **但 fallback 扫描 12780~12829 自动找真活 port** → 不影响测试

→ **建议**: 印 162 可改 e2e 用 `[System.Security.Principal.WindowsIdentity]::GetCurrent().Name -replace '.*\\','' ` 取 SID 用户名 · 与 Node 一致.
