## 哲学

### 不重写 · 只桥接

5 个模块各自完整, 本印**零修改**它们的内核, 仅在其暴露的公共 API 之上**挂载新能力**。

```
原则:
  - 每个 bridge 独立 / 可插拔 / 零侵入
  - bridge 不修改模块, 只调用模块
  - 模块下线 → bridge 自动降级 (fallback)
  - 模块升级 → bridge 自动跟随 (lazy require)
```

### 损之又损

```
+ 7 个新文件 (dao.js · E2E · 5 桥 · bridges.js 前端)
+ 3 处增量改动 (orchestrator 暴露 _hubServer · sp hook · /bridges.js)
- 0 处删除
- 0 npm 依赖
- 0 外部依赖
```

约 2200 行新代码, 全部使用 Node.js 标准库 (`http`, `https`, `fs`, `path`, `os`, `child_process`)。

## 五桥 (印227 · 04-Hub/integrations/)

### 1. `wam_loader.js` · WAM 池自动加载 (260 邮密)

`~/.wam/accounts.md` → Hub 账号池

```
启动时:    2s 后异步 autoLoad → 260 邮密 → Hub 账号池
HTTP API:  /api/wam/{load, auth-all, auth-batch, reset-exhausted, status}
特点:      幂等 (按 email 去重) · 受控并发认证 · 不阻塞启动
```

### 2. `sp_bridge.js` · 03-SP 接入 Hub directUpstream

03-SP/sp_core 五能 → Hub 所有 chat 请求

```
hook 点:   directUpstream() 中 messages → sp_core 处理 → 上游
策略:      bypass / override / prepend / append / dao / custom / usernote / invert
HTTP API:  /api/sp/{silk, dryrun, preview}
帛书加载:  7204 字 (《老子》帛书本 + 《阴符经》)
```

### 3. `github_bridge.js` · UserRouter 注入 Hub Pipeline

GitHub OAuth → 加 WS 账号 → Hub 8 阶段流水线全自动

```
用户流:    GitHub Device Flow → token → ensureFork/Gist/Pages
           → 加 WS 账号 → 写 Gist (dao.json) → 写 Secrets
           → 触发 Hub runFullPipeline → 闭环成立
HTTP API:  /api/github/{login-start, login-poll, login-paste, whoami,
                       setup, add-ws, status, logout}
CN 友好:   PAT 直接粘贴模式 (绕过 OAuth)
```

### 4. `devin_mirror_router.js` · Devin 镜像双模式

04-Hub/devin_mirror/ (88 资源 / 17MB) → /devin/

```
模式 A (passthrough):  完全官方直连, 用户用自己 cookie
模式 B (intercept · 默认): 拦截 fetch → 重定向到 Hub 网关 (用 WAM 池)
HTTP API:  /devin/* (静态) + /api/devin-mirror/{info, mode}
注入脚本:  /devin/_dao_inject.js (运行时拦截 fetch + WebSocket)
```

### 5. `dual_window.js` · 双开支持

一个 Win 账户同时跑两个 Hub 实例

```
主实例:    :7800 (Dashboard) + :7799 (Gateway)
副实例:    :7810 (Dashboard) + :7811 (Gateway)
共享:      ~/.dao/accounts.json · ~/.wam/accounts.md · 03-SP/silk/*
独立:      Pipeline / 各 VM 池 / SP 策略 / 网关健康
HTTP API:  /api/dual/{spawn, kill, status}
```

## 顶层入口

### `dao.js`

```bash
node dao.js                    # 默认 · Hub + 全部 bridges
node dao.js minimal            # 仅 Hub · 不挂 bridges
node dao.js add <em> <pw>      # CLI 加号 (全链路 4 步)
node dao.js token <em> <tok>   # CLI 加 token (直入)
node dao.js wam-load           # 一次加载 WAM 260 邮密
node dao.js wam-auth-all       # 批量四步链认证 (并发 5)
node dao.js gh-login           # GitHub Device Flow
node dao.js gh-setup <em> <pw> # 一键 GH setup + 加号
node dao.js status             # 查 Hub 健康
node dao.js dual               # 双开
node dao.js stop               # 停所有 node
node dao.js e2e                # 跑 11 项 E2E
```
