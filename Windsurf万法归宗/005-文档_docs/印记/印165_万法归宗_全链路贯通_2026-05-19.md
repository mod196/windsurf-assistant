## 二、修复记录 (v2.1 · 印165)

### 根因
- `windsurf_relay.js` 原调用 `cloud_engine.connectRpcStream` → 该函数未暴露 → HTTP 404
- `cloud_engine.chatStream` 直连 `inference.codeium.com` → 推理服务端已迁移 → HTTP 404
- 真正工作路径：LSP Cascade → 道直连器(7861) → LSP桥接

### 修复
1. **windsurf_relay.js v2.1**: 用 HTTP 直连道直连器(port 7861) 替换失效的 cloud_engine 直连
2. **chatStreamFull**: 非流式 → POST 7861 → JSON响应
3. **callWindsurfRelay streaming**: SSE透传 → 直通道直连器SSE流
4. **_getDaoSkKey()**: 从 `~/.dao/keys.json` 读取活跃 dao-sk-* key
5. **listModelsFull**: 修复 `_lazyLoadAccounts` 引用，恢复 54 Cascade + 6 Devin 全量
