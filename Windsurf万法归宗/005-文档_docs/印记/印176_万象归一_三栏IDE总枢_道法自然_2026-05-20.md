## 八 · 用户工作流 (实现一切)

### 工作流 A: 像 Devin 官网开发项目 + 存 Git

1. 浏览器打开 `http://127.0.0.1:7788`
2. **右栏** 直接看到 Devin 官网完整界面 (auth1_session 已注 · 无需登录)
3. 在右栏 Devin 创建会话、写代码、和 AI 对话
4. **左栏 → 📦 项目** 创建项目，填:
   - 项目名 / GitHub repo / 关联 sessionId / 本地工作目录 cwd
5. 点 "✓ Commit" / "↑ Push" → 直接推到自己的 GitHub 仓库
6. 点 "↗ 在Devin打开" → 右栏自动跳到该会话

### 工作流 B: 虚拟机反代 API + 公网无感使用

1. **左栏 → 🌐 反代** 配置 VM URL (例如 `https://xxx.devinapps.com`)
2. **顶栏 → [VM公网]** 切换反代模式
3. 所有 API 请求(Cascade/Devin/三方)走 VM 反代 · 设备无关
4. 在不同设备登录 GitHub 即得相同反代体验 (印175 user_router 闭环)

### 工作流 C: BYOK 多 Provider Key 池

1. **左栏 → 🔌 API** 选 Provider (6 类)
2. 添加 API Key (sk-... / ghp_... / etc)
3. 测试可达性 (HEAD baseUrl)
4. 设置模型→Provider 映射 (如 `claude*` → `anthropic`)
5. 反代核心自动按映射路由请求 → 用对应 Key

### 工作流 D: SP 提示词注入 (印169 已立)

1. **左栏 → 📝 SP** 创建/激活提示词
2. 顶栏点 "💉 注入当前SP" → 注入到右栏 Devin 下次会话
3. SP 可前置/替换/后置/dao 四种策略
