## 一、根因修复 (§5 try-catch 级联失败)

### 逆向发现
从 Playwright 测试日志反向推导：
```
[WARNING] [道] 历史劫持失败: Cannot assign to read only property 'replace' of object '[object Location]'
```
**根因**: §5 所有代码在一个 try 块中：
- `window.location.replace = function(...){}` 抛异常 → catch 捕获
- `history.pushState` 补丁永不执行
- auth 重定向 `/auth/login` 无法拦截 → 死循环

### 修复
各节独立 try-catch：
```js
// §5a: location.assign (单独 try)
try { window.location.assign = ... } catch(e) {}
// §5b: location.replace (单独 try，允许失败)  
try { window.location.replace = ... } catch(e) {}
// §5c: history.pushState/replaceState (核心·必须执行)
try {
  window.history.pushState = function(s,t,u){ if(shouldBlock(u)) return; ... };
  window.history.replaceState = ...;
  console.log('[道·印172] history.pushState/replaceState 补丁已激活');
} catch(e) {}
// §5d: History.prototype 级拦截
try { Object.defineProperty(History.prototype, ...) } catch(e) {}
```

function Ahe() {
  const e = localStorage.getItem(fg);
