2 · Windsurf Account · for VM deploy

Your Windsurf API key (from ~/.dao/accounts.json or Windsurf settings). This is what your VM uses to talk to Windsurf cloud. Stays in your browser until you copy it into the Devin task.

2.5 · Windsurf 账号底层 · 一键取 sk-* (印 64 · 4 步链)

No API key on hand? Enter your Windsurf email + password — the VM walks the login → postauth → register → status chain on your behalf and returns a fresh sk-ws-*. Endpoint must be reachable AND --allow-auth must be set on the VM (default OFF; turn on by appending DAO_ALLOW_AUTH=1 to the bootstrap env).
Privacy: password leaves your browser only to your own VM (which calls windsurf.com directly). It is never persisted unless you tick "Remember password" below.

Architecture · Fully Decentralized

┌─────────────────────────────────────┐ │ GitHub (your fork · any account) │ │ ├─ Pages → this single web UI │ │ ├─ Actions → auto-deploy on push │ │ └─ Repo → source of truth │ └────────────────┬────────────────────┘ │ visit ▼ ┌─────────────────────────────────────┐ │ Browser (your device) │ │ ├─ All config in localStorage │ │ └─ Direct HTTPS to your VM only │ └────────────────┬────────────────────┘ │ Authorization: Bearer sk-ws-proxy-* ▼ ┌─────────────────────────────────────┐ │ fleet_vm_unit.js (anywhere) │ │ ├─ Node.js zero-dep proxy │ │ ├─ OpenAI /v1 compatible │ │ └─ cloudflared tunnel (public URL) │ └────────────────┬────────────────────┘ │ Connect-RPC over HTTPS ▼ ┌─────────────────────────────────────┐ │ Windsurf Cloud │ │ └─ inference.codeium.com │ └─────────────────────────────────────┘ No central PC client. No relay server. No middleman. Each user runs their own VM. Each VM has its own account, IP, fingerprint.

API Endpoints

POST /v1/chat/completions OpenAI-compatible chat (SSE stream when stream=true) GET /v1/models Model list (54+ models · grouped by vendor) GET /quota Real-time quota (daily%/weekly%) GET /health Health + auth state (public · for probes) GET /fleet/info Unit metadata (public) Headers: Authorization: Bearer sk-ws-proxy-XXX (required when --auth-key set) Content-Type: application/json

Self-Host Steps

A. Fork https://github.com/zhouyoukang/windsurf-assistant to your account B. Settings → Pages → Source: GitHub Actions (already configured by deploy-pages.yml) C. Push any change to web/ — Actions auto-deploys to: https://<your-username>.github.io/<your-fork-name>/ D. Visit your page. Setup tab auto-detects your owner/repo. E. Provision a VM (Devin Cloud is free for limited use; or any VPS/EC2/RPi): - Generate auth-key in Setup → copy - Generate Devin Task in Deploy → paste into Devin chat - Devin executes and reports the public tunnel URL F. Paste tunnel URL into Setup → Test Connection G. API tab now shows your reverse-proxy base_url + key — ready for any OpenAI client

Zero Dependencies

Both ends require nothing except git + Node.js ≥ 18 + curl.
  • Web UI: single HTML file, no build, no npm, no CDN
  • VM unit: 5 .js files, only Node.js builtins (http/https/crypto/fs/path/os)
  • Tunnel: cloudflared single binary, optional
  • No relay server: browser → your VM, direct
